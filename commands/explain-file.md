# explain-file

## 명령어 목적

특정 파일의 역할, 주요 코드 흐름, 다른 파일과의 관계를 초보자 눈높이로 설명한다.

## 사용 상황

- Java 파일 하나가 무슨 역할인지 알고 싶을 때
- Controller, Service, Repository, Model, Dto의 차이를 이해하고 싶을 때
- Spring Boot 설정 파일의 의미를 알고 싶을 때
- 코드를 수정하기 전에 해당 파일을 먼저 이해하고 싶을 때

## 실제 프롬프트

```text
선택한 파일을 초보자도 이해할 수 있게 설명해줘.

조건:
1. 파일은 수정하지 마.
2. AGENTS.md 규칙을 먼저 확인해줘.
3. 이 파일이 프로젝트에서 어떤 역할을 하는지 설명해줘.
4. 중요한 메서드나 설정이 있다면 쉬운 말로 풀어서 설명해줘.
5. 다른 파일과 연결되는 부분이 있으면 함께 알려줘.
6. 수정이 필요해 보이는 부분이 있으면 실제 수정하지 말고 제안만 해줘.
7. 모든 설명은 한국어로 해줘.
```
