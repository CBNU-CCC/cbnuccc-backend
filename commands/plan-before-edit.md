# plan-before-edit

## 명령어 목적

파일을 수정하기 전에 필요한 변경 범위, 수정 대상, 테스트 방법을 먼저 계획으로 정리한다.

## 사용 상황

- 기능 추가나 버그 수정을 시작하기 전
- 어떤 파일을 고쳐야 할지 아직 확실하지 않을 때
- 초보자가 수정 흐름을 먼저 이해하고 싶을 때
- 큰 변경 전에 안전한 작업 순서를 잡고 싶을 때

## 실제 프롬프트

```text
아직 파일은 수정하지 말고, 먼저 수정 계획만 세워줘.

조건:
1. AGENTS.md 규칙을 먼저 확인해줘.
2. Java/Spring/Gradle 프로젝트 기준으로 분석해줘.
3. 어떤 파일을 확인했고, 어떤 파일을 수정할 가능성이 있는지 알려줘.
4. 왜 수정이 필요한지 설명해줘.
5. 기존 코드를 삭제해야 한다면 삭제 이유와 대안을 먼저 설명해줘.
6. 수정 후 실행하거나 테스트할 명령어를 알려줘.
7. 초보자도 이해할 수 있게 한국어로 설명해줘.
8. 내가 승인하기 전까지 실제 파일은 수정하지 마.
```
