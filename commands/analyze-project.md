# analyze-project

## 명령어 목적

현재 프로젝트의 기술 스택, 폴더 구조, 실행 방법, 필요한 설정을 초보자도 이해할 수 있게 분석한다.

## 사용 상황

- 처음 보는 프로젝트를 파악하고 싶을 때
- 로컬 실행 전에 어떤 준비가 필요한지 알고 싶을 때
- Java/Spring/Gradle 프로젝트 구조를 간단히 설명받고 싶을 때
- 파일을 수정하지 않고 읽기 전용으로 분석만 받고 싶을 때

## 실제 프롬프트

```text
현재 열린 프로젝트를 먼저 분석해줘.

조건:
1. 아직 파일은 절대 수정하지 마.
2. AGENTS.md 규칙을 먼저 확인해줘.
3. 이 프로젝트가 어떤 기술 스택인지 추정해줘.
4. 폴더 구조를 초보자도 이해할 수 있게 설명해줘.
5. 로컬에서 실행하려면 어떤 명령어와 설정이 필요한지 알려줘.
6. application.properties, build.gradle, Dockerfile, README.md를 우선 확인해줘.
7. 실제 수정이 필요한 부분이 있으면 수정하지 말고 제안만 해줘.
8. 모든 설명은 한국어로 해줘.
```
