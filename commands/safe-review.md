# safe-review

## 명령어 목적

프로젝트 또는 특정 변경 사항을 안전하게 리뷰하고, 버그 가능성, 보안 위험, 테스트 누락, 초보자가 놓치기 쉬운 부분을 찾아낸다.

## 사용 상황

- 코드를 수정한 뒤 문제가 없는지 확인하고 싶을 때
- Pull Request를 만들기 전에 위험한 변경이 있는지 보고 싶을 때
- Spring Security, JWT, DB, 환경변수처럼 조심해야 하는 부분을 점검하고 싶을 때
- 수정은 하지 않고 리뷰 의견만 받고 싶을 때

## 실제 프롬프트

```text
이 프로젝트를 안전하게 리뷰해줘.

조건:
1. 파일은 수정하지 마.
2. AGENTS.md 규칙을 먼저 확인해줘.
3. 버그 가능성, 보안 위험, 테스트 누락, 실행 실패 가능성을 우선순위로 봐줘.
4. Java/Spring/Gradle 프로젝트 기준으로 리뷰해줘.
5. 문제가 있으면 파일 경로와 이유를 함께 설명해줘.
6. 기존 코드를 삭제하거나 바꿔야 한다면, 실제 수정하지 말고 이유와 대안을 제안만 해줘.
7. 초보자도 이해할 수 있게 한국어로 설명해줘.
```
