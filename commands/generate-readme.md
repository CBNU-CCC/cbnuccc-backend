# generate-readme

## 명령어 목적

현재 프로젝트 구조와 설정을 바탕으로 초보자도 따라 할 수 있는 README 초안을 작성한다.

## 사용 상황

- README가 부족하거나 깨져 있을 때
- 프로젝트 소개, 기술 스택, 실행 방법, 환경변수 설명을 정리하고 싶을 때
- 새 팀원이 프로젝트를 쉽게 실행할 수 있게 문서를 만들고 싶을 때
- 기존 README를 바로 바꾸기 전에 초안을 먼저 보고 싶을 때

## 실제 프롬프트

```text
이 프로젝트에 맞는 README.md 초안을 작성해줘.

조건:
1. 아직 파일은 수정하지 마.
2. AGENTS.md 규칙을 먼저 확인해줘.
3. build.gradle, application.properties, Dockerfile, README.md를 우선 확인해줘.
4. Java/Spring/Gradle 프로젝트 기준으로 작성해줘.
5. 프로젝트 소개, 기술 스택, 폴더 구조, 환경변수, 로컬 실행 방법, 테스트/빌드 명령어를 포함해줘.
6. 민감 정보는 예시 값으로만 표시하고 실제 키를 넣지 마.
7. 초보자도 그대로 따라 할 수 있게 한국어로 작성해줘.
8. 수정이 필요하면 실제 파일을 바꾸지 말고 초안만 보여줘.
```
