# CBNU CCC Backend Server Development Guide

&nbsp;본 문서는 충북대학교 CCC 서버의 개발 가이드를 제공하는 문서입니다.

## Git 커밋 메시지 접두사

|  접두사  |                    상황                     |
| :------: | :-----------------------------------------: |
| feature  |             기능을 추가했을 때              |
| release  |        배포할 때 (_release: x.y.z_)         |
|   fix    |              기능을 수정할 때               |
|  revert  |             이전으로 되돌릴 때              |
|   docs   |      문서/주석을 수정하거나 추가할 때       |
| refactor |                리팩토링할 때                |
|  merge   | 병합 시 오류 해결할 때 (_merge: 충돌 해결_) |
|   기타   |      기타 상황에 잘 맞는 접두사를 사용      |

커밋 메세지: **(접두사): (구체적인 내용(한글))**

## Git 브랜치 전략

|  브랜치 이름   |              용도               |                                 비고                                 |
| :------------: | :-----------------------------: | :------------------------------------------------------------------: |
|    release     |             배포용              |               main -> release로 merge, 직접 수정 불가                |
|      main      |      주요 커밋 통합 브랜치      |                     본 브랜치로 merge 시 PR 필수                     |
| feature/(기능) |        해당 기능 구현용         |                              개발 전용                               |
|     hotfix     | main 브랜치 내 코드 긴급 수정용 | 코드 구현할 양이 상당히 클 것으로 예상될 시 feature 브랜치 이용 권장 |

코드는 항상 **feature/\* -> main -> release**로 향하도록 구현한다.

## 변수 및 함수명 명명 규칙

코드의 통일성을 위해 아래 규칙을 따를 것을 권장한다.

모든 이름은 CamelCase를 사용한다.

변수명은 명사여야 한다. 단, boolean 타입의 변수의 경우 의문문 형식의 이름(예: isError)을 사용할 수 있다.

함수명은 동사로 시작하며 관사 등의 중요하지 않은 단어는 생략하여 표현한다. 또한 함수명이 길더라도 축약하여 작성하지 않는다. 함수명을 작성할 때 아래 표의 권고 사항을 따른다.

|  동사  |        상황        |                   비고                    |
| :----: | :----------------: | :---------------------------------------: |
| create | 생성할 때 (CREATE) |                                           |
|  find  |  가져올 때 (READ)  | Controller를 제외한 모든 단에서 사용한다. |
|  get   |  가져올 때 (READ)  |        Controller에서만 사용한다.         |
| update | 수정할 때 (UPDATE) |                                           |
| delete | 삭제할 때 (DELETE) |                                           |
